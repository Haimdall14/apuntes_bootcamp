 Rails

Para el aprendizaje de Rails empezaremos viendo SQL es el lenguaje estándar utilizado para interactuar con bases de datos relacionales, y la mayoría de las aplicaciones web de hoy en día requieren la persistencia de datos. Rails se integra de manera muy eficiente con bases de datos relacionales como PostgreSQL, MySQL, SQLite, entre otros, y los desarrolladores deben ser capaces de escribir consultas y comprender cómo funcionan las bases de datos.

Conociendo SQL

SQL significa Structured Query Language, que se traduce al español como Lenguaje Estructurado de Consulta. Es un lenguaje de programación diseñado para administrar y manipular bases de datos relacionales.

Con SQL, se pueden crear, modificar y eliminar tablas y registros, y también se pueden buscar y recuperar datos de manera eficiente. Además, SQL permite la creación de consultas y reportes complejos que permiten analizar y visualizar grandes cantidades de datos de manera fácil y rápida.

SQL es utilizado por una gran variedad de aplicaciones y sistemas de gestión de bases de datos, incluyendo Oracle, MySQL, PostgreSQL, Microsoft SQL Server, entre otros. Es una herramienta fundamental para el almacenamiento y manejo de información en cualquier tipo de organización.

FAQ (Frequently Asked Questions)
¿Qué son las bases de datos y por qué las usamos?

Una base de datos es una colección organizada de información o datos que están almacenados en un sistema informático. Estos datos pueden ser de cualquier tipo, como texto, imágenes, audio, video, números, etc.

Las bases de datos se utilizan para almacenar y organizar grandes cantidades de información de manera estructurada y accesible. Permiten el almacenamiento eficiente y seguro de datos, lo que permite una rápida recuperación de información cuando es necesaria.

Las bases de datos son utilizadas en una amplia variedad de campos, desde el comercio electrónico y la banca, hasta la gestión de inventarios y el seguimiento de pacientes en la industria de la salud. Son esenciales para las organizaciones que manejan grandes cantidades de información y necesitan acceder a ella de manera rápida y eficiente.

Además, las bases de datos permiten la integración de múltiples aplicaciones y sistemas, lo que hace posible la automatización de procesos y la eliminación de tareas manuales repetitivas, lo que puede ahorrar tiempo y costos. En general, las bases de datos son una herramienta fundamental para la gestión y análisis de información en una amplia variedad de contextos y aplicaciones.

¿Cómo hacemos consultas (preguntas) a la base de datos?

Para hacer consultas (preguntas) a una base de datos, utilizamos el lenguaje SQL (Structured Query Language). SQL nos permite hacer consultas y recuperar información de la base de datos de manera precisa y eficiente.

Existen diferentes tipos de consultas que se pueden realizar utilizando SQL, pero algunas de las más comunes son:

  • SELECT: Recupera información de una o varias tablas en la base de datos. Se especifica qué columnas se quieren recuperar y de qué tablas, y se pueden agregar condiciones para filtrar los resultados.
  • UPDATE: Actualiza información existente en una tabla. Se especifica qué columna(s) se quiere(n) actualizar y a qué valor(es).
  • INSERT: Agrega nueva información a una tabla. Se especifica qué valores se quieren agregar a cada columna de la tabla.
  • DELETE: Elimina información de una tabla. Se especifica qué filas se quieren eliminar, utilizando condiciones para filtrar los resultados.

Para realizar consultas con SQL, necesitamos utilizar un software o herramienta que nos permita conectarnos a la base de datos y ejecutar comandos SQL. Algunos ejemplos de estas herramientas son MySQL Workbench, Microsoft SQL Server Management Studio, Oracle SQL Developer, entre otros.

¿Cómo podemos construir una base de datos y llenarla con datos?

Para construir una base de datos y llenarla con datos, podemos seguir los siguientes pasos:

  • Definir los requerimientos de la base de datos: antes de construir una base de datos, es importante entender las necesidades y requerimientos de la organización o sistema que utilizará la base de datos. Esto incluye la identificación de los datos que se necesitan almacenar, la estructura de la base de datos, y las relaciones entre las diferentes tablas.
  • Diseñar la estructura de la base de datos: una vez que se han identificado los requerimientos de la base de datos, es importante diseñar la estructura de la misma. Esto incluye la creación de tablas, la definición de campos o columnas, y la especificación de las relaciones entre las tablas.
  • Crear la base de datos: después de diseñar la estructura de la base de datos, es necesario crearla utilizando un software de gestión de bases de datos como MySQL, PostgreSQL, Microsoft SQL Server, entre otros.
  • Crear las tablas: dentro de la base de datos, es necesario crear las tablas según la estructura previamente definida. Esto incluye definir los campos o columnas, tipos de datos y restricciones.
  • Llenar la base de datos con datos: una vez que se han creado las tablas, se pueden llenar con datos utilizando comandos SQL o herramientas de importación de datos. Es importante asegurarse de que los datos ingresados sean precisos y estén en el formato adecuado.
  • Realizar pruebas: finalmente, es importante realizar pruebas exhaustivas de la base de datos para asegurarse de que funciona correctamente y cumple con los requerimientos definidos.

Con estos pasos, podemos construir una base de datos y llenarla con datos de manera eficiente y precisa.

PostgreSQL (Instalación)

PostgreSQL es un sistema de gestión de bases de datos relacionales (RDBMS) de código abierto. Ofrece una amplia gama de características y funcionalidades para la gestión de datos, y es conocido por su escalabilidad, estabilidad y seguridad.

Es una buena opción para aquellos que necesitan una base de datos confiable, escalable y con características avanzadas.

Intalación

Si tuvieras alguna consulta o dificultad con la instalación, puedes consultar la Documentación Oficial.

Debemos ingresar la siguiente secuencias de comandos en Terminal.

# Abrimos terminal WSL y escribiremos:

sudo apt update # Actualiza la lista de versiones de paquetes.
sudo apt upgrade # Actualiza nuestro repositorio de paquetes.

sudo apt install postgresql postgresql-contrib
# Se instalarán 2 paquetes necesarios.

sudo -u postgres createuser --interactive
# Ingresar la contraseña de instalación de WSL.
# Rol de Superuser? (y/n): ingresar Y (si).

Comprobamos

Para verificar que la instalación fue satisfactoria, realizamos los siguientes tests:

psql --version
# Muestra la versión actual instalada en el equipo.

sudo service postgresql start
# Inicia el servicio de PostgreSQL.

sudo -u postgres psql
# inicia sesión PostgreSQL como usuario "postgres" con permisos de superusuario.

# El Prompt cambiará a:
postgres=#
# Significa que conseguimos conectarnos a la base de datos exitosamente.

Comandos mas comunes en SQL

A continuación una pequeña relación de comandos usados frecuentemente, no está de mas buscar Documentación en la web.

  • SELECT : Recupera datos de una o más tablas.
  • INSERT : Añade datos a una tabla existente.
  • UPDATE : Modifica los datos contenidos en una tabla.
  • DELETE : Suprime todo o una parte de los datos de la tabla.
  • CREATE DATABASE : Crea una nueva base de datos externa (archivos db .4db y .4dd) en una ubicación específica.
  • USE DATABASE : Designa una base externa como base de datos actual, en otras palabras, la base a la cual se dirigirán las próximas consultas SQL en el proceso actual.
    Todos los tipos de consultas SQL concernientes: consultas incluidas en la estructura Begin SQL/End SQL, comandos SQL EXECUTE o EXECUTE SCRIPT, etc.
  • ALTER DATABASE : Activa o deshabilita las opciones SQL de la base actual para la sesión actual; Es decir para todos los usuarios y procesos hasta que la base se reinicie.
    Este comando está diseñado para permitirle deshabilitar temporalmente las opciones SQL con el fin de acelerar ciertas operaciones que consumen una gran cantidad de recursos, por ejemplo: desactivar los índices y las restricciones antes de comenzar la importación de una gran cantidad de datos puede reducir significativamente la duración de la importación.
    Tenga en cuenta que las restricciones incluyen las llaves primarias y las llaves foráneas, así como los atributos de unicidad y de nulidad.
    Si prefiere gestionar los triggers individualmente para cada tabla, debe utilizar ALTER TABLE.
  • CREATE TABLE : Utiliza para crear una tabla.
  • DROP INDEX : Elimina la base el índice existente. No puede utilizarse con los índices creados por las restricciones PRIMARY KEY o UNIQUE.
  • LOCK TABLE : permite bloquear la tabla nom_sql en el modo EXCLUSIVE o SHARE.
    En el modo EXCLUSIVE, los datos de la tabla no pueden leerse o modificarse por otra transacción.
    En el modo SHARE, los datos de la tabla pueden ser leídos por otras transacciones pero las modificaciones siguen siendo prohibidas.
  • UNLOCK TABLE : Desbloquea la tabla previamente bloqueada vía el comando LOCK TABLE. Este comando no funciona si se pasa dentro de una transacción o si se utiliza en una tabla que bloqueada por otro proceso.
  • EXECUTE IMMEDIATE : Ejecuta una expresión SQL dinámica. Representa una variable que contiene un conjunto de instrucciones SQL que se ejecutarán como un todo.
  • CREATE SCHEMA : Crea un nuevo esquema SQL en la base de datos. Puede utilizar todo nom_sql excepto por "DEFAULT_SCHEMA" y "SYSTEM_SCHEMA".
  • ALTER SCHEMA : Renombra el esquema SQL nom_sql (primer parámetro) por nom_sql (segundo parámetro).
    Sólo el Diseñador y el Administrador pueden modificar esquemas.
  • DROP SCHEMA : Elimina el esquema designado.
    Es posible borrar cualquier esquema, excepto el esquema por defecto (DEFAULT_SCHEMA) y el esquema que contiene las tablas sistema ("SYSTEM_SCHEMA"). Cuando se elimina un esquema, todas las tablas que le fueron asignadas se transfieren al esquema por defecto. Las tablas transferidas heredan los derechos de acceso del esquema por defecto.
    Si intenta eliminar un esquema que no existe o que ya se ha eliminado, se genera un error.
  • CREATE VIEW : Permite crear una vista SQL (estándar) que contiene las columnas definidas en el parámetro lista_columnas. Es necesario especificar un nombre de columna si esta columna es una función o se deriva de una operación aritmética (escalar). También es necesario especificar un nombre de columna cuando se quiere evitar tener distintas columnas con el mismo nombre (por ejemplo, durante una operación JOIN) o cuando desea utilizar un nombre de columna diferente del que se deriva.
  • DROP VIEW : Borra de la base la vista llamada nom_vista.
    Cuando se pasa la restricción IF EXISTS, el comando no hace nada y no se genera ningún error si la vista nom_vista no existe en la base.
  • REVOKE : Borra los derechos de acceso específicos asociados al esquema definido por el parámetro nom_sql.
    Cuando ejecuta este comando, asigna el pseudo-grupo de usuarios (Persona) al derecho de acceso definido.
  • REPLICATE : Replica los datos de una tabla de una base A en la de una tabla de una base B. Por convención, la base donde se ejecuta el comando se llama "base local" y la base de la cual los datos se replican se llama "base remota".
    Este comando sólo puede utilizarse en el marco de un sistema de replicación de base. Para que el sistema funcione, la replicación debe haber sido activada en la base local y en la base remota y cada tabla implicada deberá tener una llave primaria.
  • SYNCHRONIZE : Sincroniza dos tablas ubicadas en dos servidores 4D SQL diferentes. Todo cambio realizado en una de las tablas también se efectúa en el otro. El servidor 4D SQL que ejecuta el comando se llama servidor local y el otro servidor se llama servidor remoto.
    El comando SYNCHRONIZE es una combinación de dos llamadas internas al comando REPLICATE. La primera llamada replica los datos desde el servidor remoto al servidor local y la segunda efectúa la operación inversa: replicación de los datos del servidor local al servidor remoto. Las tablas a sincronizar deben estar configuradas para la replicación

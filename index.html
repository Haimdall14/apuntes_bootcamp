Configurando terminal

A continuación algunos comandos de terminal para empezar a familiarizarnos (minúsculas).

pwd = Muestra la ruta completa de la ubicación actual.
clear = Limpia/Borra la pantalla del terminal.
ls -la = Muestra todos los archivos dentro del directorio, incluidos los ocultos.
mkdir = Crear directorio (mkdir nombre_de_directorio).
cd = Cambiar de directorio (cd ruta/de/la/carpeta).
rmdir = Elimina un directorio (rmdir ruta/o/nombre/de/la/carpeta).
rm -rf = Elimina un "archivo" en cualquier directorio.
Instalando WSL

WSL (Windows Subsystem for Linux) es una característica de Windows que permite ejecutar una distribución de Linux en Windows 10 y versiones posteriores. Esto significa que puedes tener acceso a herramientas de línea de comandos de Linux y ejecutar programas de Linux directamente en Windows, sin necesidad de una máquina virtual o de instalar Linux en una partición separada.

WSL es útil para desarrolladores que trabajan con herramientas y tecnologías específicas de Linux, como Ruby on Rails, Python, Node.js, Docker y Kubernetes. Con WSL, pueden ejecutar estas herramientas y tecnologías directamente en Windows, lo que les permite desarrollar y probar aplicaciones en un entorno de desarrollo similar al de producción.

Instalando windows terminal y ohmyzsh

Windows Terminal es una aplicación de línea de comandos moderna desarrollada por Microsoft que proporciona una experiencia de terminal personalizable y potente en Windows 10. Con Windows Terminal, los usuarios pueden abrir varias pestañas en una única ventana y trabajar con varias shells, como PowerShell, CMD, WSL y más.

Oh My Zsh, por otro lado, es un framework de código abierto para la línea de comandos de Unix, que se ejecuta en sistemas operativos macOS y Linux. Ofrece una gran cantidad de características y herramientas que hacen que trabajar en la línea de comandos sea más fácil y eficiente, como autocompletado, sugerencias de comandos, temas personalizables, plugins, etc.

En conjunto, Windows Terminal y Oh My Zsh pueden proporcionar una experiencia de línea de comandos altamente personalizable y potente en Windows, con características avanzadas y herramientas de productividad para mejorar la eficiencia del usuario en el trabajo en la línea de comandos.

Instalando Ruby

Ruby es un lenguaje de programación dinámico, de alto nivel y orientado a objetos que se utiliza para desarrollar una amplia variedad de aplicaciones, desde aplicaciones web hasta scripts de automatización y herramientas de línea de comandos.

Ruby es un lenguaje de programación versátil que se puede utilizar en una amplia variedad de aplicaciones, desde aplicaciones web hasta juegos y análisis de datos.

Instalando NodeJS

Node.js se utiliza principalmente para desarrollar aplicaciones web y servidores de aplicaciones que manejan grandes cantidades de datos y solicitudes simultáneas. Al utilizar JavaScript tanto en el lado del cliente como del servidor, Node.js permite a los desarrolladores crear aplicaciones de una manera más coherente y eficiente.

Node.js también es conocido por su escalabilidad, ya que puede manejar un gran número de conexiones simultáneas con una latencia extremadamente baja. Además, tiene una gran cantidad de librerías y herramientas disponibles a través de su administrador de paquetes, npm, lo que facilita el desarrollo de aplicaciones.

Algunas de las características de Node.js incluyen la programación asíncrona, la capacidad de manejar archivos y datos en tiempo real, la capacidad de ejecutar scripts del lado del servidor, la creación de aplicaciones en tiempo real y la capacidad de crear aplicaciones híbridas móviles y de escritorio.

Instalando VSCode

VSCode es un editor de código fuente de código abierto desarrollado por Microsoft que admite múltiples lenguajes de programación y plataformas. Es una de las herramientas de desarrollo más populares utilizadas por los desarrolladores de software en la actualidad.

VSCode es altamente personalizable y extensible gracias a su gran cantidad de extensiones, lo que permite a los desarrolladores adaptar su entorno de desarrollo a sus necesidades específicas. Instalando Git

Git es un sistema de control de versiones de software de código abierto que se utiliza para registrar los cambios realizados en archivos y coordinar el trabajo en equipo en proyectos de software. Fue desarrollado originalmente por Linus Torvalds en 2005 para el desarrollo del kernel de Linux, pero desde entonces se ha convertido en una herramienta popular en muchos otros proyectos de software.

Git permite a los desarrolladores mantener un historial completo de cambios en un proyecto, rastrear quién hizo cada cambio y cuándo se realizó, y revertir a versiones anteriores si es necesario. También permite a los desarrolladores trabajar en diferentes ramas de código al mismo tiempo, lo que facilita la colaboración en equipo.


Despues de haber instalado Git abrimos Terminal y digitamos los siguientes comandos:

git --version
# Mostrará la versión de git que tiene instalada.

# Para instalar la última versión estable de Git en Ubuntu:
sudo add-apt-repository ppa:git-core/ppa
sudo apt update
sudo apt install git

git --version
# Consultamos nuevamente para confirmar que se actualizó a la ultima versión

Para finalizar con la configuracion de Git debemos registrar un "nombre de usuario", "contraseña" y "correo electrónico"; Recomiendo anotarlas en un lugar seguro, "editor de código predeterminado" (VSCode) y la "rama inicial predeterminada" (main).

git config --global user.name "Saturnino Reyes"
# Entre las comillas ingresarás tu nombre de usuario.

git config --global user.email "aquivatucorreo@mail.com"

# Los siguientes comandos ejecutarlos tal cual línea por línea
git config --global core.autocrlf input
git config --global core.safecrlf true
git config --global core.editor "code --wait"
git config --global init.defaultBranch main

Configurando Github

Ahora debemos vincular nuestra cuenta de Git con nuestra cuenta de Github, para eso podemos consultar la Documentación Oficial, o también podemos ejecutar los siguientes pasos:

Debemos abrir la terminal, una vez dentro asegurarnos de estar en home:

cd ~

Una vez estamos en home ejecutamos los siguientes comandos:

ssh-keygen -o
# (Presionar Enter) Empezamos a generar una llave Publica/Privada.
# Luego te pedirá que ingreses tu Passphrase (se recomienda dejarlo en blanco y solo presionar Enter)
# Pedirá confirmación de Passphrase (Nuevamente presionar enter)

# Creada la llave Publica/Privada nos dirigimos al directorio ssh:
cd ~/.ssh

# Dentro del directorio ssh ingresamos el comando:
ll

# Esto nos enlistará el contenido del directorio ssh
cat id_rsa.pub
# Nos abre el archivo id_rsa.pub veremos un monton de caracteres, todo eso es la llave debemos "copiarlo" de principio a fin

Nos vamos al navegador y entramos a nuestra cuenta de Github y nos dirigimos a la esquina superior derecha en la foto de nuestra cuenta y damos click, se pespliega un menu de opciones y seleccionamos Settings.

Luego en el Menu del lado izquierdo seleccionamos la opcion SSH and GPG Keys.

Hacer Click en New SSH Key.

En el campo "Title" (Título), agrega una etiqueta descriptiva para la clave nueva. Por ejemplo, si estás utilizando un portátil personal, puedes llamar a esta clave "Portátil personal".

Pega tu clave pública (la que copiamos en terminal -id_rsa.pub-) en el campo "Key".

Y por ultimo click en Add SSH key.

Regresamos a Terminal e ingresamos el siguiente comando:

ssh -T git@github.com

Puedes ver una advertencia como la siguiente:

The authenticity of host 'github.com (IP ADDRESS)' can't be established.
ECDSA key fingerprint is SHA256:+DiY3wvvV6TuJJhbpZisF/zLDA0zPMSvHdkr4UvCOqU.
Are you sure you want to continue connecting (yes/no)?

Escribimos yes (si), se nos mostrará el siguiente mensaje:

Hi USERNAME! You've successfully authenticated, but GitHub does not provide shell access.

Listo ya tenemos git vinculado con nuestra cuenta github.
